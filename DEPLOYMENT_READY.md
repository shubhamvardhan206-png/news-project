## 🚀 Quick Deploy Steps

### 1. Push to GitHub
```bash
cd ~/Desktop/news-project
git add .
git commit -m "Prepare for Render deployment"
git push origin main
```

### 2. Visit https://render.com
- Sign up with GitHub
- Authorize Render access

### 3. Create Web Service
- Dashboard → New Web Service
- Select news-project repo
- Click Connect

### 4. Configure Service
```
Name: newsproject
Environment: Python 3
Build Command: pip install -r newsproject/requirements.txt && python newsproject/manage.py collectstatic --noinput && python newsproject/manage.py migrate
Start Command: gunicorn newsproject.wsgi:application --bind 0.0.0.0:$PORT --workers 2
Plan: Free
```

### 5. Create PostgreSQL Database
```
Name: newsproject_db
Region: Same as web service
Plan: Free
```

### 6. Add Environment Variables
```
DEBUG = False
SECRET_KEY = [generate: python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"]
ALLOWED_HOSTS = newsproject.onrender.com
DATABASE_URL = [auto-set by Render]
NEWS_API_KEY = 83fad5e997474c6aa8705aad5edb636d
```

### 7. Deploy!
Click "Create Web Service" - wait 2-5 minutes

### 8. Visit Your Site
```
https://newsproject.onrender.com
```

## ⚠️ Important Notes

### What Works Great
✅ Development & testing
✅ Small traffic sites (<100 req/day)
✅ Learning & portfolio projects
✅ Auto-deployment from GitHub

### What's Limited
⚠️ Media file persistence (use S3 for production)
⚠️ Cold start time (30s on first request)
⚠️ Auto-spins down after 15 min inactivity
⚠️ Limited CPU/memory

### For Production
💡 Upgrade to paid plan ($7+/month)
💡 Use external storage for media
💡 Set up monitoring & alerts
💡 Configure backups
